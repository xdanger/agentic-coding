# 更新日志 (Changelog)

本文档记录项目的所有**显著更改**。一个更改被认为是"显著的"，如果它符合以下条件之一：

1. 引入新功能或删除现有功能
2. 修改对用户可见的行为
3. 修复关键错误
4. 更改架构或设计
5. 修改依赖关系
6. 更改项目结构

仅内部实现的修改、较小的 bug 修复、文档更新和代码优化通常不记录在此文件中。
